# XChaCha20\-Poly1305 安全文件加密工具 \- README

## 一、项目简介

这是一款基于 **XChaCha20\-Poly1305 AEAD** 算法的高性能、高安全文件加密解密工具，使用 Go 语言开发。采用流式分块处理机制，恒定内存占用，完美支持 KB 至 TB 级超大文件，兼顾加密安全性、完整性校验与运行性能，适用于个人私密文件、数据备份、涉密文件的本地加密存储。

工具全程无明文缓存、内存敏感数据自动擦除、文件最小权限管控，从算法、内存、文件权限多维度规避数据泄露、篡改、损坏风险。

## 二、核心安全特性

- **军用级加密算法**：采用 XChaCha20\-Poly1305 标准 AEAD 算法，支持加密\+完整性校验双重能力，有效抵御窃听、篡改、重放攻击

- **超大文件流式处理**：固定 64KB 缓冲区流式读写，全程无全局明文缓存，内存占用恒定，无 OOM 风险，支持 TB 级文件

- **防跨文件篡改**：将原始文件名作为关联数据（AD）绑定密文，杜绝密文被替换、跨文件非法解密

- **内存安全防护**：密钥、明文、密文等敏感内存数据使用后立即安全擦除，规避编译器优化导致的内存残留、内存dump泄露风险

- **最小权限原则**：密钥文件权限 0600、目录权限 0700，仅所有者可读写，杜绝其他用户访问

- **密钥覆盖二次确认**：检测到已有密钥文件时主动弹窗确认，防止误覆盖密钥导致密文永久失效

- **精细化完整性校验**：逐块校验密文 Tag，文件截断、篡改、密钥错误时立即终止并报错，不输出损坏明文

- **合规随机数**：基于系统安全随机源生成密钥、Nonce，无伪随机数漏洞，全局唯一 Nonce 杜绝加密漏洞

## 五、使用教程

工具使用极简，自动根据文件后缀判断加密/解密逻辑，无需额外参数。

### 1\. 文件加密

对任意文件进行加密，自动生成 `原文件.enc`（密文）和 `原文件.key`（密钥）

```Plain Text
# Windows
encrypt-tool.exe test.iso

# Linux/Mac
./encrypt-tool test.iso
```

**加密输出**：

- 密文文件：`test.iso.enc`（加密后的文件，无法直接打开）

- 密钥文件：`test.iso.key`（解密唯一凭证，务必备份）

**注意**：若目录已存在同名密钥，工具会二次确认是否覆盖，避免误操作。

### 2\. 文件解密

对 `.enc` 后缀密文文件解密，自动读取同目录下的 `.key` 密钥恢复原文件

```Plain Text
# Windows
encrypt-tool.exe test.iso.enc

# Linux/Mac
./encrypt-tool test.iso.enc
```

**解密输出**：自动还原为原始文件 `test.iso`

### 3\. 查看帮助

```Plain Text
encrypt-tool.exe
```

## 六、文件结构说明

### 密文文件（\.enc）格式

1. 文件头（24字节）：全局唯一 XChaCha20 Nonce

2. 数据块（循环结构）：
        

    - 4字节大端整数：当前密文段长度

    - 变长密文：明文加密结果 \+ Poly1305 校验 Tag

该结构支持流式逐块校验，任意文件篡改、截断都会触发解密失败，安全性极高。

### 密钥文件（\.key）

固定 32 字节 XChaCha20 安全密钥，权限严格限制为 0600，是解密的唯一凭证。

## 七、重要注意事项

> **核心警告**：**密钥文件（\.key）丢失、损坏、删除后，密文文件将永久无法解密，无任何找回方式！**
> 
> 

- 加密后请**立即备份 \.key 密钥文件**，离线存储、云端备份均可，切勿与密文放在同一目录长期存放

- 切勿手动修改、重命名 \.enc 密文文件和 \.key 密钥文件，会导致解密校验失败

- 密钥覆盖后，旧密文无法使用新密钥解密，旧密钥对应的密文永久失效

- 工具仅对文件内容加密，不加密文件名，如需隐藏文件名可自行前置处理

- 解密失败大概率为：密钥错误、密文被篡改/截断、文件传输损坏

## 八、报错说明

- `密钥损坏，需32字节`：密钥文件不完整、被修改、传输截断

- `加密文件过小，已损坏或截断`：密文文件未传输完成、被恶意修改

- `解密校验失败`：密钥不匹配、密文篡改、文件损坏

- `用户取消操作`：主动取消密钥覆盖操作
